Writes a file to the local workspace.

Usage:
- This tool will overwrite the existing file if there is one at the provided path.
- The path parameter must stay inside the workspace.
- If this is an existing file, you should read it first unless the user explicitly asked for a full rewrite.
- ALWAYS prefer editing existing files in the codebase. NEVER write new files unless explicitly required.
- NEVER proactively create documentation files (*.md) or README files unless the user explicitly asks.
- Only use emojis if the user explicitly requests them.
