Read the accumulated stdout and stderr from a long-running background process.

Usage:
- Pass a process_id previously returned by `background_process_start`.
- Use this tool to inspect whether a dev server or watcher has started successfully.
- After `background_process_start`, check this tool before assuming the service is ready. For HTTP services, wait until logs show a listening address or successful startup message before issuing requests.
- If a matching workspace service may already be running, inspect the existing process logs before starting a replacement.
- The output is observational; use `background_process_stop` to terminate the process.
