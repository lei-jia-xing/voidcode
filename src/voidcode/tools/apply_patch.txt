Apply unified diff patches to files inside the local workspace.

Usage:
- The patch parameter is required and must be a unified diff string that git apply can validate.
- Use this tool for multi-file edits, renames, deletions, or patch-shaped changes that are clearer as a diff.
- Prefer edit or multi_edit for small local replacements in a single file.
- Read the relevant files first so the patch is grounded in real context.
- Use the returned changes and affected paths as the source of truth for follow-up verification.
