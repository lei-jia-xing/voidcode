Use this tool to create and manage a structured task list for the current coding session. This helps track progress, organize complex tasks, and make work state explicit.

When to use this tool:
- Complex multistep tasks
- Non-trivial implementation work
- When the user explicitly asks for a todo list
- When the user gives multiple tasks or expands scope

When not to use this tool:
- A single straightforward task
- A trivial change that can be completed immediately
- Purely conversational or informational requests

Task states:
- pending
- in_progress
- completed
- cancelled

Keep items concrete and current. Treat data.summary as the stable result field.
